# 生产级优化决策记录

更新日期：2026-07-19

## 决策摘要

| 决策       | 选择                                                         | 理由与影响                                                                        |
| ---------- | ------------------------------------------------------------ | --------------------------------------------------------------------------------- |
| 前端方向   | 保留 Next.js 模块化单体，建立编辑式内容设计系统              | 避免重写数据闭环；以留白、字号、对齐和内容节奏替代卡片套卡片                      |
| 设计 Token | 在 `globals.css` 集中管理颜色、间距、圆角、阴影、动效和断点  | 页面不再散落任意视觉值；暗色和减少动画共用同一套语义变量                          |
| 搜索召回   | 分阶段多通道召回 + RRF 合并 + 有界语义精排                   | 全量语义计算在 20k 合成目录上过慢；先做低成本召回，再对约 180 个候选计算语义特征  |
| 硬约束     | 排序前、排序后均执行确定性硬门控                             | 语言、许可证、部署、排除条件不能被个性化或热度分数抵消                            |
| 搜索排序   | 保留可解释加权模型，增加校准、来源可信度和最低证据阈值       | 当前标注集不支持可靠训练 LTR；特征和版本继续持久化，为 LightGBM/LambdaMART 留接口 |
| 推荐队列   | 用户/日期/算法/目录版本维度预生成最多 50 个暖候选            | 首批和换批只做轻量会话重排；反馈会清除已预取的未来批次及其未真实发生的曝光        |
| 负反馈     | 按语言、复杂度、维护状态、相似主题分别限定作用域并衰减       | 防止一次误操作永久污染整个领域；“已看过”只抑制重复，不等同“不喜欢”                |
| 赞助预留   | 独立召回、独立计分、独立表和独立曝光；默认 Feature Flag 关闭 | 关闭时自然结果逐项不变；开启时最多 1 个，必须显式标注并通过相关、质量和安全门槛   |
| 本地语义   | 继续使用确定性多语言词典、哈希向量和提取式证据               | 无密钥可完整运行，但界面不把它描述成外部大模型能力                                |
| 会话安全   | HMAC 签名且带过期时间的会话，生产强制 32 位以上密钥          | 不再信任客户端可编辑的明文用户 Cookie                                             |
| 规范来源   | OAuth/CSRF 使用 `APP_URL`，不信任 Host/Forwarded Host        | 阻断 Host Header 注入；生产公网 URL 强制 HTTPS，本机回环预览允许 HTTP             |
| SEO        | 生产构建允许索引，开发构建保持 noindex                       | 公开演示可被发现，同时避免开发服务被意外索引                                      |

## 没有采用的方案

- 没有引入交叉编码器或在线 LLM 精排：60 项人工目录和 27 条查询不足以证明收益，且会破坏无密钥、低延迟和确定性要求。
- 没有把全部页面改成统一网格：发现、搜索、详情和知识库改用不同内容节奏，共享的是 Token 与交互语法。
- 没有把赞助候选混入自然评分：商业信号永远不能提高自然推荐分。
- 没有声称 PostgreSQL 已完成在线压测：当前主机没有 Docker、PostgreSQL 或 `psql`，只完成 Schema、迁移生成与静态校验。

## 可回滚边界

- 搜索权重、行为权重、探索比例和模型版本均位于配置层。
- `GPS_SPONSORSHIP_ENABLED=false` 可完全关闭赞助路径。
- 文件运行时仍保留为零依赖演示适配器；PostgreSQL Schema 不改变现有 JSON 数据格式。
- GitHub、Embedding、LLM、邮件和对象存储继续通过 Provider 接口隔离。

## 2026-07-19 跨领域与文案校正

- 冷启动不再预选科学计算或可视化；用户必须主动选择方向，避免系统先验替用户定义兴趣。
- 演示目录从 48 项扩到 60 项，新增云原生、基础设施、数据库、数据工程、机器学习、移动、游戏、3D 创作、智能家居、安全和远程协作。
- 冷启动种子按 12 个领域各取一个，再补足到 12 项，防止数据文件顺序决定首屏内容。
- 搜索词典新增上述领域的中英文概念映射，并修复 “application security” 被误判为 application 项目类型的歧义。
- 用户界面删除“本地语义”“系统理解”“搜索审计”“暖队列”等实现术语；推荐说明收敛为用途、技术、近期或长期偏好等可核验短句。
- 搜索示例改用开放式行列表，首页与搜索页不再用物理例子代表产品能力；物理仍作为平等领域保留在索引和回归测试中。

## 2026-07-19 字体、动效与重复搜索热路径

- 字体不引入外部网络依赖：中文优先使用 MiSans、HarmonyOS Sans SC、苹方和微软雅黑 UI，英文标题优先使用系统可变 Display 字体，仓库路径与代码使用 Cascadia Code/JetBrains Mono 等宽栈。这样兼顾中英文字面协调、隐私和首屏速度。
- 动效只表达加载、层级和反馈因果：卡片入场、搜索解析扫描、结果簇渐显、批次进度、按钮与标签采用 120–400 ms 的 opacity/transform 动效；触控设备不执行悬浮景深，`prefers-reduced-motion` 下压缩到 0.01 ms。
- 搜索结果先显示 4 条主要解决路线，其余每次展开 4 条；保留全部结果和聚类能力，同时降低首屏长度与认知负担。
- 进程内搜索流水线缓存以“完整结构化意图 + 模式 + 用户画像指纹 + 目录指纹 + 算法版本”为 Key，TTL 5 分钟、最多 400 项；缓存 Promise 合并同 Key 并发计算，画像、目录或版本变化自动换 Key，异常时立即删除。
- “不要 Java”一类排除语言改为规范化后的精确语言约束，不再把 Java 当作 JavaScript 子串；负向语言也不会同时进入正向语言条件。
- 移动端封面取消窄屏强制 16:9 最小高度组合，避免封面和数据来源标记超出卡片；桌面继续保持稳定 16:9 视觉。

## 2026-07-20 质量门控、双语简介与滚动重构

- 推荐资格不再只由相关性得分决定。成熟仓库必须达到 1,000 Stars；较新仓库需同时满足 100 Stars、质量 0.82、活跃维护，并由作者 5,000 关注者、200 Watchers 或 30 日新增 300 Stars 中至少一项证明外部关注。归档、Fork、镜像、缺少 README 或超过一年未 Push 的项目在排序前剔除。
- GitHub 实时同步补充 Watchers 与仓库所有者 Followers；字段为可空列，不伪造缺失值。演示 Watchers 是明确标记的确定性派生值，只服务离线闭环。
- 桌面推荐卡由纵向海报改为横向编辑式版面，目标高度约 30.5rem；简介、推荐理由和标签均限制首屏行数，详细简介按需展开。搜索条件、知识库高级工具和详情证据使用渐进披露，减少长页面滚动但保留全部功能。
- 全局语言偏好存于浏览器 `gps_locale`，同步更新文档语言；导航、推荐卡、Agent 与项目简介可即时切换。仓库原始描述仍保持来源语言，AI/本地归纳层负责统一展示语言，避免假装翻译原始证据。
- DeepSeek 仅由服务器 Provider 调用，输出通过 Zod Schema；默认 `deepseek-v4-flash`、JSON 输出、20 秒超时。无密钥、超时或格式不符时，项目简介回退到证据模板，Agent 回退到确定性站内规则。
- Agent 不能直接执行收藏、删除或账户操作，只返回经过路径白名单过滤的站内导航／搜索动作；赞助推荐仍保持独立关闭，不向 Agent 暴露商业排序控制。

## 2026-07-20 信息密度与任务型 Agent 二次校正

- 未登录首页不再使用占满视口的单一巨幅标题和叠放空卡片；首屏同时呈现产品价值、进入方式、三类跨领域项目预览与领域覆盖，桌面无需首屏滚动即可理解产品，移动端按“价值—操作—项目预览”单列展开。
- 字体栈改为系统可变 UI 字体优先，中文使用微软雅黑 UI／Noto Sans SC 等本地回退；中文标题减少负字距并降低粗度，正文行高从松散展示收敛到连续阅读尺度，不引入阻塞首屏的远程字体。
- 推荐页桌面主内容宽度限制为 70rem，卡片目标高度约 23.75rem；批次环形进度改为细轨道，窄屏隐藏右侧重复画像摘要，避免用户浏览完单个项目后还必须滚过整组次要面板。
- 冷启动步骤隐藏移动底栏与 Agent。初始化是单一任务流，悬浮入口和全局导航会遮挡“下一步”并增加误离开概率；完成后两者自动恢复。
- 移动端 Agent 入口缩为单图标触点，面板使用单一消息滚动区，不再出现“面板滚动 + 页面滚动 + 重复关闭按钮”。
- 本地 Agent 升级为 `gps-agent-rules-v2`：区分问候、趋势、订阅、个人知识库、画像管理和项目查找；对“我想做一个 Agent”补全为工具调用、工作流与可扩展插件查询，并返回搜索与趋势两个可执行入口。快捷问题点击即发送，不要求再次确认。
- 外部 Agent 请求失败时只显示本地化恢复提示，不把 Origin、Provider 或内部错误原样暴露给用户；开发日志仍保留诊断信息。
