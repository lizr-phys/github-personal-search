## 缓存策略

| 对象                 | Key                                  | TTL/边界                  | 失效与降级                                                             |
| -------------------- | ------------------------------------ | ------------------------- | ---------------------------------------------------------------------- |
| 查询意图             | 规范化查询 + 解析版本                | 1 h LRU                   | 词典/版本变化自然换 Key；失败回退规则解析                              |
| 项目检索文档         | 仓库 ID + `dataUpdatedAt`            | 目录进程生命周期，LRU     | 元数据刷新改变版本；缺字段使用可用证据                                 |
| 本地语义向量         | 文档版本                             | 延迟生成，LRU             | Provider 失败时词法/Topic/RRF 继续工作                                 |
| GitHub 元数据/README | URL + Token 隔离                     | ETag/条件请求和客户端 TTL | 304 复用；超时/限流返回缓存并标记过期                                  |
| 暖推荐队列           | 用户 + 日期 + 算法 + 目录版本        | 当日、最多 50             | 反馈、清数据、目录/算法版本变化失效；不足时快速补充                    |
| 搜索结果             | 意图 + 模式 + 画像 + 目录 + 算法版本 | 5 min LRU，最多 400 项    | 画像/目录/版本变化自动换 Key；Promise 合并并发；失败删除；不跨画像复用 |
| 趋势/详情            | 仓库快照/目录更新时间                | 随快照版本                | 缺新快照显示数据时间，不伪装实时                                       |

## 数据库说明

Schema 具有曝光、搜索结果、算法版本、快照、向量和赞助独立索引；迁移静态检查通过。当前主机没有 PostgreSQL，因此无法提供真实 `EXPLAIN ANALYZE`、连接池和锁竞争数字，这一项保持为未执行而非“通过”。
