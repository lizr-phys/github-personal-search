# GPS 生产级优化最终质量报告

完成日期：2026-07-20

## 结论

本轮已完成实际代码、迁移、设计系统、算法、缓存、安全与测试改造。P0 演示闭环在六类浏览器/设备上最终 12/12 通过；iOS 快速连续反馈在修复触控滚动后单独复测 1/1 通过。类型、Lint、53 项单元/集成测试、生产构建、算法离线评测、20k 规模测试、混合压力和依赖审计均通过。当前本机生产测试服务为 `http://localhost:3200`。

不能宣布“所有生产基础设施验收完成”：当前主机没有 Docker、PostgreSQL、Redis CLI，因而实际 Compose 启动、在线迁移、`EXPLAIN ANALYZE`、Redis/数据库断线恢复未执行；Vercel 预览部署在 10 分钟内超时，没有可交付的公网 URL。

## 已完成优化

### 前端设计与页面布局

- 集中设计 Token：纸面/墨色/状态色、间距、字级、圆角、阴影、内容宽度、动效和断点。
- 发现页改为编辑式内容流，批次主题、10 点进度、稳定封面、证据、反馈和批次结束节点形成完整节奏。
- 搜索页加入主搜索、建议、自然语言理解摘要、主要条件 Chip、渐进高级条件和解决路线聚类。
- 详情页改为技术项目杂志式首屏，推荐证据、能力、成本和趋势优先，低优先元数据后移。
- 知识库围绕状态、合集、笔记、标签、关系、历史和学习记录组织。
- 修复桌面主按钮覆盖、品牌文本换行、iOS WebKit 输入提交、表单可访问名称和弱文本对比度。
- 推荐卡采用桌面横向编辑式结构与移动紧凑操作轨道；搜索、详情和知识库的次级内容改为渐进披露。
- 增加全局中英文切换、证据型统一语言项目简介和右下角 GPS Agent；DeepSeek 不可用时自动降级到本地证据模板与站内规则。

### 交互动画与可访问性

- 高频动效 120–220 ms、页面动效 200–400 ms，使用 opacity/transform；减少动画模式近乎归零。
- 支持键盘跳转链接、焦点状态、暗色、触控、最小点击目标、移动底部导航和离线错误反馈。
- 截图等待渐显动画完成，避免把中间帧作为交付证据。

### 搜索算法

- 查询规范化、技术实体/Topic 映射、中英同义词、缩写、错别字、排除条件、置信度和 TTL 缓存。
- 词法、Topic、画像、趋势、关系和探索多通道召回；RRF 合并后只对有界候选做语义精排。
- 硬约束在粗排和最终排序双重门控；增加最低证据阈值，不再用弱相关项目填满结果。
- 分模式控制探索、时效和多样性；特征、来源与算法版本继续可追踪。

### 推荐算法

- 每用户/日期/算法/目录版本预生成最多 50 个暖候选；首批与换批只做轻量会话重排。
- 将评分从重复扫描交互改为预聚合信号；显式负反馈和“已看过”采用不同作用域与半衰期。
- 当前反馈会使已预取未来批次和未发生曝光失效，下一批重新排序。
- 配额与 MMR 共同保证组织、语言、框架、用途多样性和中等探索比例。
- 增加独立质量资格层：成熟项目至少 1,000 Stars；新项目需同时满足活跃维护、质量和作者 Followers／Watchers／近期增长证据。60 项演示目录中 58 项合格、2 项被阻断，合格项目才可进入推荐流。

### 广告预留

- `GPS_SPONSORSHIP_ENABLED` 默认关闭；关闭时自然列表逐项不变。
- 赞助候选独立召回/计分/曝光/点击/屏蔽表，须通过最低相关、质量和安全门槛，每批最多 1 个并明确标“赞助项目”。

### 性能与稳定性

- 引入压缩文档索引、LRU、查询意图缓存、延迟语义向量、有界精排、暖队列批次缓存和并发去重边界。
- 重复搜索增加画像/目录/算法版本隔离的 5 分钟流水线缓存与 Promise 合并；专项复测重复搜索 P95 从 46.24 ms 降到 23.03 ms。
- 20k 合成目录冷搜索从 17.493 s 降到 3.672 s，暖搜索从 2.719 s 降到 1.152 s，RSS 峰值从 903 MB 降到 461.5 MB。
- 生产 API 换批 P95 8.91 ms、暖搜索 P95 35.66 ms、10 并发搜索 P95 218.27 ms。

### 安全

- HMAC 签名/过期会话，生产强密钥；OAuth/CSRF 使用规范 `APP_URL`，不信任 Host。
- 5xx 响应脱敏并保留 request ID。
- 图片代理限制 HTTPS 白名单、DNS 私网、重定向、大小、超时和 MIME，拒绝 SVG。
- Webhook 先验原始字节 HMAC、1 MiB 上限、坏 JSON 400。
- 基线 2 个 moderate 依赖漏洞已清零。

## 修复的问题

- 首次全矩阵出现 Safari 跳转链接策略差异和 iOS 搜索提交禁用，修复后完整矩阵 12/12。
- 修复图片/文字弱对比、关系/趋势/画像表单无可访问名称。
- 修复重复 onboarding 知识库项、预取批次不响应反馈、负反馈作用域过宽。
- 修复主按钮样式覆盖和品牌副标题同行显示。
- 修复测试截图捕捉动画中间帧造成的错误视觉证据。
- 修复排除 Java 时误伤 JavaScript/TypeScript、搜索路线一次性铺满和移动封面数据标记越界；均加入单元或浏览器回归证据。
- 修复项目简介快速刷批触发 429、Agent 将“找知识库”误判为打开收藏库、模型动作白名单接受相似恶意路径，以及 iOS 连续反馈时滚动稳定性不足。
- 修复未登录首页信息稀疏、中文标题字距过紧、推荐页重复次要滚动和移动 Agent 入口过宽；首屏现在同时展示价值、权限、跨领域样例和覆盖范围。
- 修复本地 Agent 只重复能力说明的问题：短 Agent 开发需求会被扩展为工具调用、工作流和插件条件，并提供搜索与趋势动作。
- 修复 iOS 冷启动“下一步”被 Agent／底部导航拦截，以及知识库弱文本 4.41:1 的 AA 对比度缺陷；修复后 Chrome+iOS 核心和无障碍 4/4 通过。

## 未解决问题与风险

1. 移动 Lighthouse LCP 2.616 s，比 2.5 s 目标高约 0.116 s；桌面满足目标。
2. 当前主机没有 Docker/PostgreSQL/Redis，Compose、实际迁移、`EXPLAIN ANALYZE`、连接池/锁竞争和基础设施断线恢复未实测。
3. Vercel CLI 部署和 `whoami` 均超时，且主机没有 Bash 运行无认证回退脚本；没有公网预览 URL。
4. 公网多实例必须把进程内限流迁移到 Redis/网关，并为 CSP 完成 nonce 化后移除 `unsafe-inline`。
5. 测试链接默认是 60 个跨领域演示项目；搜索页“拉取 GitHub”可按需抓取公开仓库，但真实 OAuth Stars 回调仍需要项目所有者配置 GitHub OAuth 凭据。
6. 20k 压测为合法合成元数据，不能代替 PostgreSQL/pgvector 上的真实数据分布。
7. 邮件完成应用内预览与签名反馈，没有验证真实域名 SMTP 的退信、投诉和送达率。
8. 聊天中曾公开的 DeepSeek 密钥未写入仓库也未用于调试，必须轮换后以 `DEEPSEEK_API_KEY` 配置；本轮已验证无密钥降级、Schema、超时和站内动作白名单。
