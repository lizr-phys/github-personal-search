# GPS v1.0 实施计划

## 交付目标

在单个模块化 Next.js 应用中交付可本地运行的 GPS MVP。默认演示模式不依赖 GitHub、邮件或模型密钥，但仍覆盖冷启动、10 个一批的推荐、会话反馈重排、中英文搜索、项目详情、知识库、趋势、订阅、邮件预览和隐私控制。配置 GitHub OAuth 后启用真实登录、公开 Stars 导入与按需仓库同步。

## 垂直切片

1. 基础运行：锁定依赖；建立 PostgreSQL/pgvector、Redis、迁移、种子与演示用户；建立应用壳、响应式导航、状态组件和测试框架。
2. 搜索闭环：实现 `QueryParser`、`QueryExpander`、词法/本地语义/趋势/图关系召回、可解释加权排序、MMR、多路线聚类、查询理解编辑和项目详情。
3. 推荐闭环：实现兴趣初始化、30—50 个暖候选、固定首批 10 个、曝光记忆、“再来 10 个”、当前会话反馈重排、30 日去重和证据化解释。
4. 知识库：实现收藏、合集、状态、标签、笔记、学习记录、历史搜索和项目关系，并将深层行为回写画像。
5. GitHub 集成：实现最小权限 OAuth、加密 Token、Stars 导入、Repository/README/Topics/Languages/Releases 获取、ETag 缓存、退避和配额记录。
6. 趋势与订阅：实现 1/7/30 日确定性热度、筛选、语义订阅、30 日投递去重和本地邮件预览。
7. 质量与交付：完成隐私删除、限流/校验、结构化日志、算法回放、单元/集成/E2E、类型、Lint、构建、迁移、Docker Compose 和关键页面截图。

## 完成定义

- 演示用户在 3 分钟内完成初始化，并看到 10 个不重复且至少覆盖 3 个簇的项目。
- 正/负反馈写入持久层，撤销可用，且下一批的项目或顺序发生可验证变化。
- 中文自然语言查询能扩展英文技术概念；结构化意图可编辑；结果提供综合、精确、灵感、最新四种模式和解决路线聚类。
- 收藏/学习/运行/复现/使用/放弃等状态能在知识库检索并参与画像。
- 无外部密钥时所有核心页面和流程仍可运行，并显著标注“演示数据”。
- 标准检查命令成功：`pnpm typecheck`、`pnpm lint`、`pnpm test`、`pnpm test:integration`、`pnpm test:e2e`、`pnpm build`、`pnpm db:migrate`、`pnpm db:seed`。

## 风险闸门

- 仓库事实不得由模型自由生成；描述、能力、维护和解释只由种子证据或 GitHub/README 数据派生。
- 排序权重集中在版本化配置，不出现在页面组件。
- 任何反馈先记录曝光与算法版本，再更新画像；“已看过”只抑制重复。
- 邮件和推荐在质量不足时允许少于上限，不用低质量项目补齐。
- 真实 GitHub 失败时回退到本地索引，并返回数据时间、错误类型和剩余配额。

## 执行结果

Slice 1—7 的演示闭环、算法、页面、接口、测试和文档均已完成；类型、Lint、18 个单元测试、8 个集成测试、核心 E2E、生产构建和 25 表迁移一致性检查通过。当前机器缺少 Docker CLI/PostgreSQL，Compose 健康与在线迁移/种子未执行，详见 `docs/test-results.md` 与 `docs/known-limitations.md`。
